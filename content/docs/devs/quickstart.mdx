> Get a local Mapas Culturais instance running in minutes using Docker Compose.

This guide walks you through cloning the RedeMapas repository and starting all services locally. The entire stack runs in Docker — no manual dependency installation required.

## Prerequisites

* [Docker](https://docs.docker.com/get-docker/) with Docker Compose v2 (`docker compose`, not `docker-compose`)
* [Git](https://git-scm.com/)

## Starting the local environment

<Steps>
  <Step title="Clone the repository">
    ```bash  theme={null}
    git clone https://github.com/redemapas/mapas.git
    cd mapas
    ```
  </Step>

  <Step title="Start the containers">
    ```bash  theme={null}
    docker compose up -d
    ```

    This starts three services: the PHP application, PostgreSQL with PostGIS, and MailPit for email capture. Database migrations run automatically on first boot via the container entrypoint.

    <Note>
      The application container waits for PostgreSQL to pass its health check before starting. On the first run, pulling images and running migrations may take a minute or two.
    </Note>
  </Step>

  <Step title="Access the application">
    Open [http://localhost:8080](http://localhost:8080) in your browser.
  </Step>

  <Step title="Check your email (optional)">
    MailPit captures all outbound email from the application. Open [http://localhost:8025](http://localhost:8025) to view captured messages — useful for testing registration flows, password resets, and notifications.
  </Step>

  <Step title="Connect to PostgreSQL (optional)">
    The database is exposed on `localhost:5432`. Connect with any PostgreSQL client using the credentials below, or use the `dev/psql.sh` wrapper script.

    | Parameter | Value       |
    | --------- | ----------- |
    | Host      | `localhost` |
    | Port      | `5432`      |
    | Database  | `mapas`     |
    | User      | `mapas`     |
    | Password  | `mapas-dev` |
  </Step>
</Steps>

## Services at a glance

| Service     | URL                                            | Description                                 |
| ----------- | ---------------------------------------------- | ------------------------------------------- |
| Application | [http://localhost:8080](http://localhost:8080) | Main web interface                          |
| MailPit     | [http://localhost:8025](http://localhost:8025) | Email capture UI                            |
| PostgreSQL  | `localhost:5432`                               | Database (user: `mapas`, pass: `mapas-dev`) |

## After the stack is up

Once the containers are running, these are the next guides you will usually need:

* [Docker Compose](/mnt/data/code/redemapas/manual/content/docs/devs/deploy/docker.mdx) for HTTPS with Caddy, container shell access, PsySH, PostgreSQL access, restores, and asset rebuild commands
* [Testing](/mnt/data/code/redemapas/manual/content/docs/devs/development/testing.mdx) for PHPUnit usage, test structure, builders, and directors
* [Architecture](/mnt/data/code/redemapas/manual/content/docs/devs/development/architecture.mdx) for stack, runtime flow, and workspace layout

If you are actively changing frontend code, the most common next command is:

```bash  theme={null}
dev/pnpm.sh -C src run watch
```

## Local verification checklist

After booting the stack or changing infrastructure-related code, verify these points before moving on:

* `http://localhost:8080` loads the application
* `https://mapas.localhost:8443` works if you are testing PWA or Web Push flows
* MailPit receives outbound messages at `http://localhost:8025`
* `dev/psql.sh` connects successfully and the `mapas` database is present
* the test suite starts without environment errors
* frontend assets rebuild normally through `dev/pnpm.sh -C src run watch`
