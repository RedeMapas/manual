---
title: API de Inscrições
---
> Endpoints da API para gerenciar inscrições em oportunidades no Mapas Culturais.

Uma **inscrição** é uma candidatura enviada por um agente para uma oportunidade. As inscrições percorrem um ciclo de vida do rascunho ao envio, podem ser avaliadas e receber um status, como aprovada, suplente ou não aprovada. Os dados da inscrição são privados e só podem ser vistos pelo proponente e pelos gestores da oportunidade.

Caminho base: `/api/registration/`

<Warning>
  Os dados de inscrição são privados. Todos os endpoints exigem autenticação. Requisições sem autenticação não retornam registros de inscrição. Use um token JWT válido no cabeçalho `Authorization` e defina `MapasSDK-REQUEST: true`.
</Warning>

## Endpoints

| Método   | Caminho                        | Descrição                     |
| -------- | --------------------------- | ------------------------------- |
| `GET`    | `/api/registration/find`    | Listar ou buscar registrations    |
| `GET`    | `/api/registration/findOne` | Buscar um único registration      |
| `POST`   | `/registration/index`       | Criar um novo registration       |
| `PATCH`  | `/registration/single/{id}` | Atualizar um existente registration |
| `DELETE` | `/registration/single/{id}` | Delete a registration           |

***

## GET /api/registration/find

Retorna uma lista paginada de registrations. Only returns registrations the authenticated user has permission to view.

### Parâmetros de consulta

**Parâmetros de controle** (prefixados com `@`)

<ParamField query="@select" type="string">
  Lista de campos separada por vírgula para incluir na resposta.

  Exemplo: `@select=id,number,status,opportunity,owner`
</ParamField>

<ParamField query="@page" type="integer" default="1">
  Número da página para resultados paginados. Começa em `1`.
</ParamField>

<ParamField query="@limit" type="integer" default="10">
  Número máximo de resultados por página. Use `-1` para desativar a paginação, com cautela em conjuntos grandes.
</ParamField>

<ParamField query="@order" type="string">
  Expressão de ordenação. Formato: `campo DIR`, em que `DIR` é `ASC` ou `DESC`.

  Exemplo: `@order=sentTimestamp DESC`
</ParamField>

<ParamField query="@or" type="integer">
  Quando definido como `1`, combina todos os parâmetros de filtro com lógica `OR`, em vez do `AND` padrão.
</ParamField>

**Parâmetros de filtro**

<ParamField query="id" type="string">
  Filtra por registration ID. Suporta operadores: `EQ(n)`, `IN(a,b,c)`.

  Exemplo: `id=EQ(123456)`
</ParamField>

<ParamField query="number" type="string">
  Filtra por registration number (the human-readable reference code, e.g., `ABC123456`).

  Exemplo: `number=EQ(ABC123456)`
</ParamField>

<ParamField query="status" type="integer">
  Filtra por registration status. See status values below.

  Exemplo: `status=EQ(1)` for sent/pending registrations.
</ParamField>

<ParamField query="opportunity" type="string">
  Filtra por opportunity using entity reference syntax.

  Exemplo: `opportunity=EQ(@Opportunity:15)` returns all registrations for opportunity ID 15.
</ParamField>

<ParamField query="owner" type="string">
  Filtra por the applicant agent using entity reference syntax.

  Exemplo: `owner=EQ(@Agent:42)` returns registrations belonging to agent ID 42.
</ParamField>

<ParamField query="category" type="string">
  Filtra por registration category (if the opportunity uses categories).

  Exemplo: `category=EQ(Pintura)`
</ParamField>

<ParamField query="proponentType" type="string">
  Filtra por tipo de proponente.

  Exemplo: `proponentType=EQ(individual)`
</ParamField>

### Resposta

Retorna um array JSON de registration objects.

<ResponseField name="id" type="integer" required>
  Identificador único de registration identifier (randomly generated integer).
</ResponseField>

<ResponseField name="number" type="string" required>
  Human-readable registration reference number (e.g., `ABC123456`). Includes a configurable prefix.
</ResponseField>

<ResponseField name="status" type="integer" required>
  Registration status code. See status values below.
</ResponseField>

<ResponseField name="category" type="string">
  The category chosen by the applicant (if the opportunity uses categories).
</ResponseField>

<ResponseField name="proponentType" type="string">
  Tipo de proponente escolhido pela pessoa proponente.
</ResponseField>

<ResponseField name="range" type="string">
  The range/tier chosen by the applicant (if the opportunity uses ranges).
</ResponseField>

<ResponseField name="opportunity" type="object">
  The opportunity this registration belongs to.

  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      Opportunity ID.
    </ResponseField>

    <ResponseField name="name" type="string">
      Opportunity name.
    </ResponseField>

    <ResponseField name="singleUrl" type="string">
      URL to the opportunity's detail page.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="owner" type="object">
  The agent that submitted this registration.

  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      Agent ID.
    </ResponseField>

    <ResponseField name="name" type="string">
      Agent name.
    </ResponseField>

    <ResponseField name="singleUrl" type="string">
      URL to the agent's profile page.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="agentRelations" type="array">
  List of related agents (co-authors, institutions, etc.) and their relation status.
</ResponseField>

<ResponseField name="spaceRelation" type="object">
  The space linked to this registration (if the opportunity requires a space).

  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      Space relation ID.
    </ResponseField>

    <ResponseField name="status" type="integer">
      Relation status. Negative values indicate pending authorization.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="files" type="object">
  Attached files keyed by file group name. Only visible to the applicant and opportunity managers.
</ResponseField>

<ResponseField name="createTimestamp" type="string">
  Data e hora em ISO 8601 when the registration was created.
</ResponseField>

<ResponseField name="sentTimestamp" type="string">
  Data e hora em ISO 8601 when the registration was submitted. Null while still in draft.
</ResponseField>

<ResponseField name="updateTimestamp" type="string">
  Data e hora em ISO 8601 of the last update.
</ResponseField>

<ResponseField name="evaluationResultValue" type="mixed">
  The consolidated evaluation result value. Only visible to users with the appropriate permission.
</ResponseField>

<ResponseField name="evaluationResultString" type="string">
  Human-readable string representation of the evaluation result.
</ResponseField>

<ResponseField name="singleUrl" type="string">
  URL to the registration detail/view page.
</ResponseField>

## GET /api/registration/findOne

Retorna um único registration. Aceita os mesmos parâmetros de filtro de `find`. Retorna o primeiro resultado encontrado.

```bash  theme={null}
curl "https://instance.example.com/api/registration/findOne?@select=id,number,status,opportunity,owner&id=EQ(987654)" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true"
```

## POST /registration/index

Cria um novo registration (draft). Exige autenticação. The authenticated user's agent becomes the registration owner.

### Corpo da requisição

| Campo           | Tipo    | Obrigatório | Descrição                                           |
| --------------- | ------- | -------- | ----------------------------------------------------- |
| `opportunityId` | integer | Sim      | The ID of the opportunity to apply to                 |
| `category`      | string  | Não       | Category selection (if the opportunity requires it)   |
| `proponentType` | string  | Não       | Tipo de proponente, se a oportunidade exigir          |
| `range`         | string  | Não       | Range/tier selection (if the opportunity requires it) |

```bash  theme={null}
curl -X POST "https://instance.example.com/registration/index" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -H "Content-Tipo: application/json" \
  -d '{
    "opportunityId": 15,
    "category": "Pintura"
  }'
```

<Note>
  A newly created registration has status `0` (draft). To submit it, use the `send` action or update the status to `1` via `save`.
</Note>

## POST /registration/single/\{id\}

Updates an existing registration. Requires authentication and ownership.

<Note>
  Once a registration is sent (status `1`), most fields become locked. Contact the opportunity manager to enable editable fields after submission.
</Note>

```bash  theme={null}
curl -X POST "https://instance.example.com/registration/single/{id}" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -H "Content-Tipo: application/json" \
  -d '{
    "id": 987654,
    "category": "Escultura"
  }'
```

***

## DELETE /registration/single/\{id\}

Deletes a registration. Only draft registrations can be deleted by the applicant. Requires authentication and ownership.

```bash  theme={null}
curl -X DELETE "https://instance.example.com/registration/single/{id}" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -d 'id=987654'
```

## Status values

Registrations use a distinct set of status codes from other entities:

| Value | Constant             | Meaning                                          |
| ----- | -------------------- | ------------------------------------------------ |
| `0`   | `STATUS_DRAFT`       | Rascunho — not yet submitted, editable by applicant |
| `1`   | `STATUS_SENT`        | Submitted / Pending evaluation                   |
| `2`   | `STATUS_INVALID`     | Invalid — rejected for procedural reasons        |
| `3`   | `STATUS_NOTAPPROVED` | Not selected                                     |
| `8`   | `STATUS_WAITLIST`    | Waitlisted                                       |
| `10`  | `STATUS_APPROVED`    | Selected / Approved                              |

<Note>
  Registration statuses differ from entity statuses used by agents, spaces, events, projects, and opportunities. Do not mix them.
</Note>

## Tips

* Only the applicant and opportunity managers can view registration data. Unauthenticated requests return empty results.
* Filter all registrations for an opportunity with `opportunity=EQ(@Opportunity:id)`. This requires opportunity manager permissions.
* Applicants can find their own registrations with `owner=EQ(@Agent:id)`.
* Campo data from custom registration form fields (type `field_*`) is included in the response when the user has view access.
* When `publishedRegistrations` is `true` on the parent opportunity, approved and waitlisted registrations become publicly visible.
