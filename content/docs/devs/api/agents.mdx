> Endpoints da API para gerenciar agentes (indivíduos e coletivos) no Mapas Culturais.

Um **agente** representa uma pessoa ou coletivo cultural cadastrado na plataforma, como artistas, produtores e gestores culturais. Agents are the primary ownership entity for spaces, events, projects, and opportunities.

Caminho base: `/api/agent/`

<Info>
  Operações de leitura são públicas. Operações de escrita (`create`, `save`, `delete`) exigem autenticação com token JWT. Veja o [guia de autenticação](/docs/devs/auth/providers) para detalhes.
</Info>

## Endpoints

| Método   | Caminho                 | Descrição              |
| -------- | -------------------- | ------------------------ |
| `GET`    | `/api/agent/find`    | Listar ou buscar agents    |
| `GET`    | `/api/agent/findOne` | Buscar um único agent      |
| `POST`   | `/agent/index`       | Criar um novo agent       |
| `PATCH`  | `/agent/single/{id}` | Atualizar um existente agent |
| `DELETE` | `/agent/single/{id}` | Excluir um agent          |

***

## GET /api/agent/find

Retorna uma lista paginada de agents. Suporta filtros, ordenação, seleção de campos e busca textual.

### Parâmetros de consulta

**Parâmetros de controle** (prefixados com `@`)

<ParamField query="@select" type="string">
  Lista de campos separada por vírgula para incluir na resposta. Use isso para limitar o tamanho do payload. Supports dot notation for related objects (e.g., `owner.name`).

  Exemplo: `@select=id,name,email,type`
</ParamField>

<ParamField query="@page" type="integer" default="1">
  Número da página para resultados paginados. Começa em `1`.
</ParamField>

<ParamField query="@limit" type="integer" default="10">
  Número máximo de resultados por página. Use `-1` para desativar a paginação, com cautela em conjuntos grandes.
</ParamField>

<ParamField query="@order" type="string">
  Expressão de ordenação. Formato: `campo DIR`, em que `DIR` é `ASC` ou `DESC`.

  Exemplo: `@order=name ASC`
</ParamField>

<ParamField query="@keyword" type="string">
  Palavra-chave de busca textual. Pesquisa nos campos de nome e descrição.
</ParamField>

<ParamField query="@or" type="integer">
  Quando definido como `1`, combina todos os parâmetros de filtro com lógica `OR`, em vez do `AND` padrão.
</ParamField>

**Parâmetros de filtro**

<ParamField query="id" type="string">
  Filtra por agent ID. Suporta operadores: `EQ(n)`, `IN(a,b,c)`, `BET(min,max)`, `!EQ(n)`.

  Exemplo: `id=EQ(42)` or `id=BET(100,200)`
</ParamField>

<ParamField query="name" type="string">
  Filtra por agent name. Suporta operadores: `LIKE(pattern)`, `ILIKE(pattern)` (case-insensitive).

  Exemplo: `name=ILIKE(maria*)` matches names starting with "maria".
</ParamField>

<ParamField query="status" type="integer">
  Filtra por publication status.

  | Value | Meaning   |
  | ----- | --------- |
  | `1`   | Publicado |
  | `0`   | Rascunho     |
  | `-2`  | Arquivado  |
  | `-9`  | Desabilitado  |
  | `-10` | Lixeira     |

  Exemplo: `status=EQ(1)`
</ParamField>

<ParamField query="type" type="integer">
  Filtra por agent type. Common values: `1` (Individual), `2` (Collective).

  Exemplo: `type=EQ(1)`
</ParamField>

<ParamField query="owner" type="string">
  Filtra por owner agent using entity reference syntax.

  Exemplo: `owner=EQ(@Agent:1)` returns agents owned by agent ID 1.
</ParamField>

<ParamField query="_geoLocation" type="string">
  Filtra por geographic proximity. Format: `GEONEAR(longitude,latitude,radius_meters)`.

  Exemplo: `_geoLocation=GEONEAR(-46.647,-23.541,700)`
</ParamField>

### Resposta

Retorna um array JSON de agent objects. The fields returned depend on `@select`.

<ResponseField name="id" type="integer" required>
  Identificador único de agent identifier.
</ResponseField>

<ResponseField name="name" type="string" required>
  Agent display name.
</ResponseField>

<ResponseField name="type" type="integer">
  Agent type. `1` = Individual, `2` = Collective.
</ResponseField>

<ResponseField name="shortDescription" type="string">
  Descrição curta (up to 400 characters).
</ResponseField>

<ResponseField name="longDescription" type="string">
  Descrição longa completa.
</ResponseField>

<ResponseField name="location" type="object">
  Coordenadas geográficas. Only returned when `publicLocation` is true or the requester has permission to view private data.

  <Expandable title="properties">
    <ResponseField name="latitude" type="number">
      Latitude coordinate.
    </ResponseField>

    <ResponseField name="longitude" type="number">
      Longitude coordinate.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="status" type="integer">
  Status de publicação. See status values above.
</ResponseField>

<ResponseField name="createTimestamp" type="string">
  Data e hora em ISO 8601 when the agent was created.
</ResponseField>

<ResponseField name="updateTimestamp" type="string">
  Data e hora em ISO 8601 of the last update.
</ResponseField>

<ResponseField name="owner" type="object">
  The parent agent (owner). For a root agent, this is the user's profile agent.

  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      ID do agente proprietário.
    </ResponseField>

    <ResponseField name="name" type="string">
      Nome do agente proprietário.
    </ResponseField>
  </Expandable>
</ResponseField>

### Exemplos

<CodeGroup>
  ```bash List published agents theme={null}
  curl "https://instance.example.com/api/agent/find?@select=id,name,email&status=EQ(1)&@limit=20"
  ```

  ```bash Search by keyword with pagination theme={null}
  curl "https://instance.example.com/api/agent/find?@select=id,name,shortDescription&@keyword=teatro&@page=2&@limit=10"
  ```

  ```bash Filtra por type and order by name theme={null}
  curl "https://instance.example.com/api/agent/find?@select=id,name,type&type=EQ(2)&@order=name%20ASC"
  ```

  ```bash Filter agents by ID range excluding one theme={null}
  curl "https://instance.example.com/api/agent/find?@select=id,name&id=AND(BET(100,200),!EQ(150))"
  ```

  ```bash Geo-proximity search theme={null}
  curl "https://instance.example.com/api/agent/find?@select=id,name,location&_geoLocation=GEONEAR(-46.6475,-23.5413,700)"
  ```
</CodeGroup>

## GET /api/agent/findOne

Retorna um único agent. Aceita os mesmos parâmetros de filtro de `find`. Retorna o primeiro resultado encontrado.

```bash  theme={null}
curl "https://instance.example.com/api/agent/findOne?@select=id,name,shortDescription&id=EQ(42)"
```

## POST /agent/index

Cria um novo agent. Exige autenticação.

<Warning>
  Operações de escrita use URLs **without** the `/api/` prefix. Include a JWT token in the `authorization` header and `MapasSDK-REQUEST: true` to receive a JSON response.
</Warning>

### Corpo da requisição

| Campo              | Tipo    | Obrigatório | Descrição                                     |
| ------------------ | ------- | -------- | ----------------------------------------------- |
| `name`             | string  | Sim      | Agent name                                      |
| `type`             | integer | Sim      | Agent type (`1` = Individual, `2` = Collective) |
| `shortDescription` | string  | Sim      | Descrição curta (máx. 400 caracteres)         |
| `longDescription`  | string  | Não      | Descrição completa                            |
| `location`         | array   | Não       | `[longitude, latitude]`                         |
| `endereco`         | string  | Não       | Street address                                  |
| `terms`            | object  | Não       | Taxonomy terms, e.g. `{"area": ["Music"]}`      |

```bash  theme={null}
curl -X POST "https://instance.example.com/agent/index" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -d 'name=Maria da Silva&type=1&shortDescription=Atriz e produtora cultural.'
```

## PATCH /agent/single/\{id\}

Partially updates an existing agent. Requires authentication and ownership (or admin role).

```bash  theme={null}
curl -X PATCH "https://instance.example.com/agent/single/42" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -d 'shortDescription=Updated description.'
```

***

## DELETE /agent/single/\{id\}

Soft-deletes an agent (moves to trash). Requires authentication and ownership (or admin role).

<Note>
  The user profile agent cannot be deleted while the user account is active.
</Note>

```bash  theme={null}
curl -X DELETE "https://instance.example.com/agent/single/42" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true"
```

## Status values

| Value | Constant          | Meaning                         |
| ----- | ----------------- | ------------------------------- |
| `1`   | `STATUS_ENABLED`  | Publicado and publicly visible  |
| `0`   | `STATUS_DRAFT`    | Rascunho, visible only to owner    |
| `-2`  | `STATUS_ARCHIVED` | Arquivado                        |
| `-9`  | `STATUS_DISABLED` | Desabilitado by admin               |
| `-10` | `STATUS_TRASH`    | Deleted (soft delete, in trash) |

## Tips

* Always use `@select` to specify only the fields you need. Omitting it returns the full object, which is slower.
* Use `@limit` and `@page` for paginated access. The default limit is `10`.
* Operator syntax is case-insensitive: `EQ(1)` and `eq(1)` are equivalent.
* To filter by related entity, use the `@Agent:id` syntax, e.g. `owner=EQ(@Agent:5)`.
