> Endpoints da API para gerenciar eventos culturais no Mapas Culturais.

Um **evento** representa uma atividade cultural com uma ou mais ocorrências agendadas em espaços. Eventos pertencem a um agente (proprietário) e podem opcionalmente estar ligados a um projeto. As ocorrências definem quando e onde o evento acontece.

Caminho base: `/api/event/`

<Info>
  Operações de leitura são públicas. Operações de escrita (`create`, `save`, `delete`) exigem autenticação com token JWT. Veja o [guia de autenticação](/docs/devs/auth/providers) para detalhes.
</Info>

## Endpoints

| Método   | Caminho                 | Descrição              |
| -------- | -------------------- | ------------------------ |
| `GET`    | `/api/event/find`    | Listar ou buscar events    |
| `GET`    | `/api/event/findOne` | Buscar um único event      |
| `POST`   | `/event/index`       | Criar um novo event       |
| `PATCH`  | `/event/single/{id}` | Atualizar um existente event |
| `DELETE` | `/event/single/{id}` | Excluir um event          |

***

## GET /api/event/find

Retorna uma lista paginada de events. Suporta filtros, ordenação, seleção de campos e busca textual.

### Parâmetros de consulta

**Parâmetros de controle** (prefixados com `@`)

<ParamField query="@select" type="string">
  Lista de campos separada por vírgula para incluir na resposta. Supports dot notation for nested objects and nested field selection for occurrences.

  Exemplo: `@select=id,name,occurrences.{id,space.{name},rule}`
</ParamField>

<ParamField query="@page" type="integer" default="1">
  Número da página para resultados paginados. Começa em `1`.
</ParamField>

<ParamField query="@limit" type="integer" default="10">
  Número máximo de resultados por página. Use `-1` para desativar a paginação, com cautela em conjuntos grandes.
</ParamField>

<ParamField query="@order" type="string">
  Expressão de ordenação. Formato: `campo DIR`, em que `DIR` é `ASC` ou `DESC`.

  Exemplo: `@order=name ASC`
</ParamField>

<ParamField query="@keyword" type="string">
  Palavra-chave de busca textual. Pesquisa nos campos de nome e descrição.
</ParamField>

<ParamField query="@or" type="integer">
  Quando definido como `1`, combina todos os parâmetros de filtro com lógica `OR`, em vez do `AND` padrão.
</ParamField>

**Parâmetros de filtro**

<ParamField query="id" type="string">
  Filtra por event ID. Suporta operadores: `EQ(n)`, `IN(a,b,c)`, `BET(min,max)`, `!EQ(n)`.

  Exemplo: `id=BET(100,200)`
</ParamField>

<ParamField query="name" type="string">
  Filtra por event name. Suporta operadores: `LIKE(pattern)`, `ILIKE(pattern)` (case-insensitive).

  Exemplo: `name=ILIKE(festival*)`
</ParamField>

<ParamField query="status" type="integer">
  Filtra por publication status.

  | Value | Meaning   |
  | ----- | --------- |
  | `1`   | Publicado |
  | `0`   | Rascunho     |
  | `-2`  | Arquivado  |
  | `-9`  | Desabilitado  |
  | `-10` | Lixeira     |

  Exemplo: `status=EQ(1)`
</ParamField>

<ParamField query="type" type="integer">
  Filtra por event type. Tipo IDs depend on instance configuration.

  Exemplo: `type=EQ(1)`
</ParamField>

<ParamField query="project" type="string">
  Filtra por associated project using entity reference syntax.

  Exemplo: `project=EQ(@Project:4)` returns events linked to project ID 4.
</ParamField>

<ParamField query="owner" type="string">
  Filtra por owner agent using entity reference syntax.

  Exemplo: `owner=EQ(@Agent:42)`
</ParamField>

### Resposta

Retorna um array JSON de event objects. The fields returned depend on `@select`.

<ResponseField name="id" type="integer" required>
  Identificador único de event identifier.
</ResponseField>

<ResponseField name="name" type="string" required>
  Event display name.
</ResponseField>

<ResponseField name="type" type="integer">
  Event type identifier.
</ResponseField>

<ResponseField name="shortDescription" type="string">
  Descrição curta (up to 400 characters).
</ResponseField>

<ResponseField name="longDescription" type="string">
  Descrição longa completa.
</ResponseField>

<ResponseField name="occurrences" type="array">
  List of scheduled occurrences for this event.

  <Expandable title="occurrence properties">
    <ResponseField name="id" type="integer">
      Occurrence identifier.
    </ResponseField>

    <ResponseField name="startsOn" type="string">
      Start date of the occurrence (ISO 8601).
    </ResponseField>

    <ResponseField name="endsOn" type="string">
      End date of the occurrence (ISO 8601).
    </ResponseField>

    <ResponseField name="startsAt" type="string">
      Start time of the occurrence.
    </ResponseField>

    <ResponseField name="endsAt" type="string">
      End time of the occurrence.
    </ResponseField>

    <ResponseField name="frequency" type="string">
      Recurrence rule (e.g., `once`, `weekly`, `monthly`).
    </ResponseField>

    <ResponseField name="rule" type="object">
      Full recurrence rule object. Contains a `description` field with a human-readable summary.
    </ResponseField>

    <ResponseField name="space" type="object">
      The space where the occurrence takes place.

      <Expandable title="properties">
        <ResponseField name="id" type="integer">Space ID.</ResponseField>
        <ResponseField name="name" type="string">Space name.</ResponseField>
        <ResponseField name="location" type="object">Space coordinates (latitude, longitude).</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="project" type="object">
  The project this event is linked to (if any).

  <Expandable title="properties">
    <ResponseField name="id" type="integer">Project ID.</ResponseField>
    <ResponseField name="name" type="string">Project name.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="status" type="integer">
  Status de publicação. See status values below.
</ResponseField>

<ResponseField name="createTimestamp" type="string">
  Data e hora em ISO 8601 when the event was created.
</ResponseField>

<ResponseField name="updateTimestamp" type="string">
  Data e hora em ISO 8601 of the last update.
</ResponseField>

<ResponseField name="owner" type="object">
  O agente proprietário deste event.

  <Expandable title="properties">
    <ResponseField name="id" type="integer">ID do agente proprietário.</ResponseField>
    <ResponseField name="name" type="string">Nome do agente proprietário.</ResponseField>
  </Expandable>
</ResponseField>

### Exemplos

<CodeGroup>
  ```bash List published events theme={null}
  curl "https://instance.example.com/api/event/find?@select=id,name,shortDescription&status=EQ(1)&@limit=20"
  ```

  ```bash Search events by keyword theme={null}
  curl "https://instance.example.com/api/event/find?@select=id,name,shortDescription&@keyword=festival"
  ```

  ```bash Get events with occurrences including space and recurrence rule theme={null}
  curl "https://instance.example.com/api/event/find?@select=id,name,occurrences.{id,space.{name},rule}&id=BET(100,200)"
  ```

  ```bash Get events with taxonomy terms filtered by language theme={null}
  curl "https://instance.example.com/api/event/find?@select=id,name,terms&term:linguagem=LIKE(Cinema)&id=BET(100,200)"
  ```

  ```bash Get all events belonging to a project theme={null}
  curl "https://instance.example.com/api/event/find?@select=id,name,location&project=EQ(@Project:4)"
  ```

  ```bash Get events with file URLs theme={null}
  curl "https://instance.example.com/api/event/find?@files=header.header,avatar.avatarBig&@select=id,name,shortDescription&id=BET(100,200)"
  ```
</CodeGroup>

## GET /api/event/findOne

Retorna um único event. Aceita os mesmos parâmetros de filtro de `find`. Retorna o primeiro resultado encontrado.

```bash  theme={null}
curl "https://instance.example.com/api/event/findOne?@select=id,name,occurrences&id=EQ(101)"
```

## POST /event/index

Cria um novo event. Exige autenticação.

<Warning>
  Include a JWT token in the `Authorization` header. Add `MapasSDK-REQUEST: true` to receive JSON responses instead of HTTP redirects on write operations.
</Warning>

### Corpo da requisição

| Campo              | Tipo    | Obrigatório | Descrição                                     |
| ------------------ | ------- | -------- | ----------------------------------------------- |
| `name`             | string  | Sim      | Event name                                      |
| `type`             | integer | Não       | Event type ID (defaults to `1`)                 |
| `shortDescription` | string  | Sim      | Descrição curta (máx. 400 caracteres)         |
| `longDescription`  | string  | Não      | Descrição completa                            |
| `projectId`        | integer | Não       | ID of the project to associate the event with   |
| `terms`            | object  | Não       | Taxonomy terms, e.g. `{"linguagem": ["Music"]}` |

<Note>
  To link an event to a project, the authenticated user must have control over that project. If not, a workflow request is created for the project owner to approve.
</Note>

```bash  theme={null}
curl -X POST "https://instance.example.com/event/index" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -H "Content-Tipo: application/json" \
  -d '{
    "name": "Festival de Jazz",
    "type": 1,
    "shortDescription": "Festival anual de jazz ao ar livre.",
    "projectId": 4
  }'
```

## POST /event/single/\{id\}

Updates an existing event. Requires authentication and ownership (or admin role).

```bash  theme={null}
curl -X POST "https://instance.example.com/event/single/{id}" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -H "Content-Tipo: application/json" \
  -d '{
    "id": 101,
    "shortDescription": "Updated description."
  }'
```

***

## DELETE /event/single/\{id\}

Soft-deletes an event. Requires authentication and ownership (or admin role).

```bash  theme={null}
curl -X DELETE "https://instance.example.com/event/single/{id}" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -d 'id=101'
```

## Status values

| Value | Constant          | Meaning                                           |
| ----- | ----------------- | ------------------------------------------------- |
| `1`   | `STATUS_ENABLED`  | Publicado and publicly visible                    |
| `0`   | `STATUS_DRAFT`    | Rascunho, visible only to owner and project managers |
| `-2`  | `STATUS_ARCHIVED` | Arquivado                                          |
| `-9`  | `STATUS_DISABLED` | Desabilitado by admin                                 |
| `-10` | `STATUS_TRASH`    | Deleted (soft delete, in trash)                   |

## Tips

* Use `@select=id,name,occurrences.{id,space.{name},rule}` to retrieve structured occurrence data with space and recurrence info in a single request.
* The `rule.description` field inside each occurrence contains a human-readable date/time summary.
* Filtra por taxonomy with `term:linguagem=LIKE(Cinema)` syntax where `linguagem` is the taxonomy slug.
* Project managers can also publish events within their project.
