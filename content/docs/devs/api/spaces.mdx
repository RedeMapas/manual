> Endpoints da API para gerenciar espaços culturais no Mapas Culturais.

Um **espaço** representa um local físico ou virtual onde eventos e atividades culturais acontecem, como teatros, galerias, centros culturais e museus. Espaços pertencem a agentes e podem sediar eventos.

Caminho base: `/api/space/`

<Info>
  Operações de leitura são públicas. Operações de escrita (`create`, `save`, `delete`) exigem autenticação com token JWT. Veja o [guia de autenticação](/docs/devs/auth/providers) para detalhes.
</Info>

## Endpoints

| Método   | Caminho                 | Descrição              |
| -------- | -------------------- | ------------------------ |
| `GET`    | `/api/space/find`    | Listar ou buscar spaces    |
| `GET`    | `/api/space/findOne` | Buscar um único space      |
| `POST`   | `/space/index`       | Criar um novo space       |
| `PATCH`  | `/space/single/{id}` | Atualizar um existente space |
| `DELETE` | `/space/single/{id}` | Delete a space           |

***

## GET /api/space/find

Retorna uma lista paginada de spaces. Suporta filtros, ordenação, seleção de campos e busca textual.

### Parâmetros de consulta

**Parâmetros de controle** (prefixados com `@`)

<ParamField query="@select" type="string">
  Lista de campos separada por vírgula para incluir na resposta. Supports dot notation for related objects (e.g., `owner.name`).

  Exemplo: `@select=id,name,address,location`
</ParamField>

<ParamField query="@page" type="integer" default="1">
  Número da página para resultados paginados. Começa em `1`.
</ParamField>

<ParamField query="@limit" type="integer" default="10">
  Número máximo de resultados por página. Use `-1` para desativar a paginação, com cautela em conjuntos grandes.
</ParamField>

<ParamField query="@order" type="string">
  Expressão de ordenação. Formato: `campo DIR`, em que `DIR` é `ASC` ou `DESC`.

  Exemplo: `@order=name ASC`
</ParamField>

<ParamField query="@keyword" type="string">
  Palavra-chave de busca textual. Pesquisa nos campos de nome e descrição.
</ParamField>

<ParamField query="@or" type="integer">
  Quando definido como `1`, combina todos os parâmetros de filtro com lógica `OR`, em vez do `AND` padrão.
</ParamField>

**Parâmetros de filtro**

<ParamField query="id" type="string">
  Filtra por space ID. Suporta operadores: `EQ(n)`, `IN(a,b,c)`, `BET(min,max)`, `!EQ(n)`.

  Exemplo: `id=EQ(10)` or `id=IN(1,2,3)`
</ParamField>

<ParamField query="name" type="string">
  Filtra por space name. Suporta operadores: `LIKE(pattern)`, `ILIKE(pattern)` (case-insensitive).

  Exemplo: `name=ILIKE(teatro*)` matches names starting with "teatro".
</ParamField>

<ParamField query="status" type="integer">
  Filtra por publication status.

  | Value | Meaning   |
  | ----- | --------- |
  | `1`   | Publicado |
  | `0`   | Rascunho     |
  | `-2`  | Arquivado  |
  | `-9`  | Desabilitado  |
  | `-10` | Lixeira     |

  Exemplo: `status=EQ(1)`
</ParamField>

<ParamField query="type" type="integer">
  Filtra por space type. Tipo IDs vary by instance configuration.

  Exemplo: `type=EQ(5)`
</ParamField>

<ParamField query="public" type="boolean">
  Filtra por whether the space is public.

  Exemplo: `public=EQ(true)`
</ParamField>

<ParamField query="owner" type="string">
  Filtra por the owning agent using entity reference syntax.

  Exemplo: `owner=EQ(@Agent:42)` returns spaces owned by agent ID 42.
</ParamField>

<ParamField query="_geoLocation" type="string">
  Filtra por geographic proximity. Format: `GEONEAR(longitude,latitude,radius_meters)`.

  Exemplo: `_geoLocation=GEONEAR(-46.6475,-23.5413,1000)`
</ParamField>

### Resposta

Retorna um array JSON de space objects. The fields returned depend on `@select`.

<ResponseField name="id" type="integer" required>
  Identificador único de space identifier.
</ResponseField>

<ResponseField name="name" type="string" required>
  Space display name.
</ResponseField>

<ResponseField name="type" type="integer">
  Space type identifier. Defined by instance configuration.
</ResponseField>

<ResponseField name="shortDescription" type="string">
  Descrição curta (up to 400 characters).
</ResponseField>

<ResponseField name="longDescription" type="string">
  Descrição longa completa.
</ResponseField>

<ResponseField name="public" type="boolean">
  Whether the space is publicly accessible.
</ResponseField>

<ResponseField name="location" type="object">
  Coordenadas geográficas of the space.

  <Expandable title="properties">
    <ResponseField name="latitude" type="number">
      Latitude coordinate.
    </ResponseField>

    <ResponseField name="longitude" type="number">
      Longitude coordinate.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="status" type="integer">
  Status de publicação. See status values below.
</ResponseField>

<ResponseField name="createTimestamp" type="string">
  Data e hora em ISO 8601 when the space was created.
</ResponseField>

<ResponseField name="updateTimestamp" type="string">
  Data e hora em ISO 8601 of the last update.
</ResponseField>

<ResponseField name="owner" type="object">
  O agente proprietário deste space.

  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      ID do agente proprietário.
    </ResponseField>

    <ResponseField name="name" type="string">
      Nome do agente proprietário.
    </ResponseField>
  </Expandable>
</ResponseField>

### Exemplos

<CodeGroup>
  ```bash Search spaces by keyword theme={null}
  curl "https://instance.example.com/api/space/find?@select=id,name,address&@keyword=teatro"
  ```

  ```bash List published spaces with owner information theme={null}
  curl "https://instance.example.com/api/space/find?@select=id,name,owner.name&status=EQ(1)&@limit=20"
  ```

  ```bash Geo-proximity search for spaces theme={null}
  curl "https://instance.example.com/api/space/find?@select=id,name,location&_geoLocation=GEONEAR(-46.6475,-23.5413,1000)"
  ```

  ```bash Filter spaces by owner agent theme={null}
  curl "https://instance.example.com/api/space/find?@select=id,name&owner=EQ(@Agent:42)"
  ```

  ```bash Select space name and nested owner name theme={null}
  curl "https://instance.example.com/api/space/find?@select=id,name,owner.name"
  ```
</CodeGroup>

## GET /api/space/findOne

Retorna um único space. Aceita os mesmos parâmetros de filtro de `find`. Retorna o primeiro resultado encontrado.

```bash  theme={null}
curl "https://instance.example.com/api/space/findOne?@select=id,name,shortDescription,location&id=EQ(10)"
```

## POST /space/index

Cria um novo space. Exige autenticação.

<Warning>
  Include a JWT token in the `Authorization` header. Add `MapasSDK-REQUEST: true` to receive JSON responses instead of HTTP redirects on write operations.
</Warning>

### Corpo da requisição

| Campo              | Tipo    | Obrigatório | Descrição                                  |
| ------------------ | ------- | -------- | -------------------------------------------- |
| `name`             | string  | Sim      | Space name                                   |
| `type`             | integer | Sim      | Space type ID                                |
| `shortDescription` | string  | Sim      | Descrição curta (máx. 400 caracteres)      |
| `longDescription`  | string  | Não      | Descrição completa                          |
| `public`           | boolean | Não       | Whether the space is publicly accessible     |
| `location`         | array   | Não       | `[longitude, latitude]`                      |
| `endereco`         | string  | Não       | Street address                               |
| `terms`            | object  | Não       | Taxonomy terms, e.g. `{"area": ["Theater"]}` |

```bash  theme={null}
curl -X POST "https://instance.example.com/space/index" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -H "Content-Tipo: application/json" \
  -d '{
    "name": "Centro Cultural Exemplo",
    "type": 5,
    "shortDescription": "Espaço dedicado a artes cênicas.",
    "public": true,
    "location": ["-46.685684", "-23.540402"],
    "endereco": "Av. Paulista, 100"
  }'
```

## POST /space/single/\{id\}

Updates an existing space. Requires authentication and ownership (or admin role).

```bash  theme={null}
curl -X POST "https://instance.example.com/space/single/{id}" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -H "Content-Tipo: application/json" \
  -d '{
    "id": 10,
    "shortDescription": "Updated description."
  }'
```

***

## DELETE /space/single/\{id\}

Soft-deletes a space (moves to trash). Requires authentication and ownership (or admin role).

```bash  theme={null}
curl -X DELETE "https://instance.example.com/space/single/{id}" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -d 'id=10'
```

## Status values

| Value | Constant          | Meaning                         |
| ----- | ----------------- | ------------------------------- |
| `1`   | `STATUS_ENABLED`  | Publicado and publicly visible  |
| `0`   | `STATUS_DRAFT`    | Rascunho, visible only to owner    |
| `-2`  | `STATUS_ARCHIVED` | Arquivado                        |
| `-9`  | `STATUS_DISABLED` | Desabilitado by admin               |
| `-10` | `STATUS_TRASH`    | Deleted (soft delete, in trash) |

## Tips

* Use `@select=id,name,owner.name` to include nested related object fields with a single request.
* Geo-proximity queries require the space to have a non-zero location set.
* Always use `@select` to limit the fields returned for better performance.
* A space can have a parent space (e.g., a room within a building). Use `parent` to filter by parent space ID.
