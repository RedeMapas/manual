title: API de Oportunidades
---
> Endpoints da API para gerenciar oportunidades (editais, chamadas e processos seletivos) no Mapas Culturais.

Uma **oportunidade** é um edital, chamada, prêmio ou qualquer processo seletivo que aceite inscrições. Oportunidades são associadas a uma entidade proprietária (agente, espaço, evento ou projeto), têm período definido de inscrição e suportam fluxos de avaliação em múltiplas fases.

Caminho base: `/api/opportunity/`

<Info>
  Operações de leitura são públicas. Operações de escrita (`create`, `save`, `delete`) exigem autenticação com token JWT. Veja o [guia de autenticação](/docs/devs/auth/providers) para detalhes.
</Info>

## Endpoints

| Método   | Caminho                       | Descrição                    |
| -------- | -------------------------- | ------------------------------ |
| `GET`    | `/api/opportunity/find`    | Listar ou buscar opportunities   |
| `GET`    | `/api/opportunity/findOne` | Buscar um único opportunity      |
| `POST`   | `/opportunity/index`       | Criar um novo opportunity       |
| `PATCH`  | `/opportunity/single/{id}` | Atualizar um existente opportunity |
| `DELETE` | `/opportunity/single/{id}` | Excluir um opportunity          |

***

## GET /api/opportunity/find

Retorna uma lista paginada de opportunities. Suporta filtros, ordenação, seleção de campos e busca textual.

### Parâmetros de consulta

**Parâmetros de controle** (prefixados com `@`)

<ParamField query="@select" type="string">
  Lista de campos separada por vírgula para incluir na resposta. Supports dot notation for related objects (e.g., `owner.name`).

  Exemplo: `@select=id,name,registrationFrom,registrationTo,publishedRegistrations`
</ParamField>

<ParamField query="@page" type="integer" default="1">
  Número da página para resultados paginados. Começa em `1`.
</ParamField>

<ParamField query="@limit" type="integer" default="10">
  Número máximo de resultados por página. Use `-1` para desativar a paginação, com cautela em conjuntos grandes.
</ParamField>

<ParamField query="@order" type="string">
  Expressão de ordenação. Formato: `campo DIR`, em que `DIR` é `ASC` ou `DESC`.

  Exemplo: `@order=registrationFrom DESC`
</ParamField>

<ParamField query="@keyword" type="string">
  Palavra-chave de busca textual. Pesquisa nos campos de nome e descrição.
</ParamField>

<ParamField query="@or" type="integer">
  Quando definido como `1`, combina todos os parâmetros de filtro com lógica `OR`, em vez do `AND` padrão.
</ParamField>

**Parâmetros de filtro**

<ParamField query="id" type="string">
  Filtra por opportunity ID. Suporta operadores: `EQ(n)`, `IN(a,b,c)`, `BET(min,max)`, `!EQ(n)`.

  Exemplo: `id=EQ(15)`
</ParamField>

<ParamField query="name" type="string">
  Filtra por opportunity name. Suporta operadores: `LIKE(pattern)`, `ILIKE(pattern)` (case-insensitive).

  Exemplo: `name=ILIKE(edital*)`
</ParamField>

<ParamField query="status" type="integer">
  Filtra por publication status.

  | Value | Meaning   |
  | ----- | --------- |
  | `1`   | Publicado |
  | `0`   | Rascunho     |
  | `-2`  | Arquivado  |
  | `-9`  | Desabilitado  |
  | `-10` | Lixeira     |

  Exemplo: `status=EQ(1)`
</ParamField>

<ParamField query="type" type="integer">
  Filtra por opportunity type. Tipo IDs depend on instance configuration.

  Exemplo: `type=EQ(1)`
</ParamField>

<ParamField query="owner" type="string">
  Filtra por owner agent using entity reference syntax.

  Exemplo: `owner=EQ(@Agent:7)`
</ParamField>

<ParamField query="registrationFrom" type="string">
  Filtra por registration open date. Supports date comparison operators.

  Exemplo: `registrationFrom=GTE(2024-01-01)`
</ParamField>

<ParamField query="registrationTo" type="string">
  Filtra por registration close date.

  Exemplo: `registrationTo=LTE(2024-12-31)`
</ParamField>

<ParamField query="publishedRegistrations" type="boolean">
  Filtra por whether results have been published.

  Exemplo: `publishedRegistrations=EQ(true)`
</ParamField>

### Resposta

Retorna um array JSON de opportunity objects. The fields returned depend on `@select`.

<ResponseField name="id" type="integer" required>
  Identificador único de opportunity identifier.
</ResponseField>

<ResponseField name="name" type="string" required>
  Opportunity display name.
</ResponseField>

<ResponseField name="type" type="integer">
  Opportunity type identifier. Defined by instance configuration.
</ResponseField>

<ResponseField name="shortDescription" type="string">
  Descrição curta / introduction text.
</ResponseField>

<ResponseField name="longDescription" type="string">
  Descrição longa completa.
</ResponseField>

<ResponseField name="registrationFrom" type="string">
  Data e hora em ISO 8601 when the registration period opens.
</ResponseField>

<ResponseField name="registrationTo" type="string">
  Data e hora em ISO 8601 when the registration period closes. Must be greater than or equal to `registrationFrom`.
</ResponseField>

<ResponseField name="publishedRegistrations" type="boolean">
  Whether the selection results have been published publicly.
</ResponseField>

<ResponseField name="registrationCategories" type="array">
  List of registration category options available to applicants.
</ResponseField>

<ResponseField name="registrationProponentTypes" type="array">
  List of valid proponent types (e.g., individual, collective, institution).
</ResponseField>

<ResponseField name="autoPublish" type="boolean">
  Whether results are published automatically at the configured timestamp.
</ResponseField>

<ResponseField name="publishTimestamp" type="string">
  Scheduled auto-publish datetime (ISO 8601).
</ResponseField>

<ResponseField name="status" type="integer">
  Status de publicação. See status values below.
</ResponseField>

<ResponseField name="createTimestamp" type="string">
  Data e hora em ISO 8601 when the opportunity was created.
</ResponseField>

<ResponseField name="updateTimestamp" type="string">
  Data e hora em ISO 8601 of the last update.
</ResponseField>

<ResponseField name="owner" type="object">
  O agente proprietário deste opportunity.

  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      ID do agente proprietário.
    </ResponseField>

    <ResponseField name="name" type="string">
      Nome do agente proprietário.
    </ResponseField>
  </Expandable>
</ResponseField>

### Exemplos

<CodeGroup>
  ```bash List published opportunities theme={null}
  curl "https://instance.example.com/api/opportunity/find?@select=id,name,registrationFrom,registrationTo&status=EQ(1)&@limit=20"
  ```

  ```bash Search opportunities by keyword theme={null}
  curl "https://instance.example.com/api/opportunity/find?@select=id,name,shortDescription&@keyword=edital%20cultura"
  ```

  ```bash Filter open opportunities (registration still active) theme={null}
  curl "https://instance.example.com/api/opportunity/find?@select=id,name,registrationTo&status=EQ(1)&registrationTo=GTE(2024-08-01)"
  ```

  ```bash Get opportunities with published results theme={null}
  curl "https://instance.example.com/api/opportunity/find?@select=id,name,publishedRegistrations&publishedRegistrations=EQ(true)"
  ```

  ```bash Filtra por owner agent ordered by registration close date theme={null}
  curl "https://instance.example.com/api/opportunity/find?@select=id,name,registrationTo&owner=EQ(@Agent:7)&@order=registrationTo%20DESC"
  ```
</CodeGroup>

## GET /api/opportunity/findOne

Retorna um único opportunity. Aceita os mesmos parâmetros de filtro de `find`. Retorna o primeiro resultado encontrado.

```bash  theme={null}
curl "https://instance.example.com/api/opportunity/findOne?@select=id,name,shortDescription,registrationFrom,registrationTo&id=EQ(15)"
```

## POST /opportunity/index

Cria um novo opportunity. Exige autenticação.

<Warning>
  Include a JWT token in the `Authorization` header. Add `MapasSDK-REQUEST: true` to receive JSON responses instead of HTTP redirects on write operations.
</Warning>

### Corpo da requisição

| Campo                        | Tipo    | Obrigatório | Descrição                                                                    |
| ---------------------------- | ------- | -------- | ------------------------------------------------------------------------------ |
| `name`                       | string  | Sim      | Opportunity name                                                               |
| `type`                       | integer | Sim      | Opportunity type ID                                                            |
| `shortDescription`         | string  | Sim      | Descrição curta                                                              |
| `longDescription`          | string  | Não      | Descrição completa                                                           |
| `registrationFrom`           | string  | Não       | Registration open date (`YYYY-MM-DD HH:MM`)                                    |
| `registrationTo`             | string  | Não       | Registration close date (`YYYY-MM-DD HH:MM`). Must be >= `registrationFrom`.   |
| `ownerEntity`                | integer | Sim      | ID of the entity this opportunity belongs to (agent, space, event, or project) |
| `registrationCategories`     | array   | Não       | List of category options                                                       |
| `registrationProponentTypes` | array   | Não      | Lista de tipos de proponente válidos                                         |
| `terms`                      | object  | Não       | Taxonomy terms                                                                 |

```bash  theme={null}
curl -X POST "https://instance.example.com/opportunity/index" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -H "Content-Tipo: application/json" \
  -d '{
    "name": "Edital de Apoio a Projetos Culturais",
    "type": 1,
    "shortDescription": "Chamada pública para projetos de artes visuais.",
    "registrationFrom": "2024-06-01 00:00",
    "registrationTo": "2024-08-31 23:59",
    "ownerEntity": 7
  }'
```

## POST /opportunity/single/\{id\}

Updates an existing opportunity. Requires authentication and ownership (or admin role).

```bash  theme={null}
curl -X POST "https://instance.example.com/opportunity/single/{id}" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -H "Content-Tipo: application/json" \
  -d '{
    "id": 15,
    "shortDescription": "Updated description."
  }'
```

***

## DELETE /opportunity/single/\{id\}

Soft-deletes an opportunity. Requires authentication and ownership (or admin role).

<Note>
  Deleting an opportunity will also remove associated registrations and evaluation data.
</Note>

```bash  theme={null}
curl -X DELETE "https://instance.example.com/opportunity/single/{id}" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -d 'id=15'
```

## Status values

| Value | Constant              | Meaning                                                  |
| ----- | --------------------- | -------------------------------------------------------- |
| `1`   | `STATUS_ENABLED`      | Publicado and accepting registrations (if within period) |
| `0`   | `STATUS_DRAFT`        | Rascunho, visible only to owner                             |
| `-1`  | `STATUS_PHASE`        | Phase opportunity (sub-phase of a multi-phase process)   |
| `-2`  | `STATUS_ARCHIVED`     | Arquivado                                                 |
| `-9`  | `STATUS_DISABLED`     | Desabilitado by admin                                        |
| `-10` | `STATUS_TRASH`        | Deleted (soft delete, in trash)                          |
| `-20` | `STATUS_APPEAL_PHASE` | Appeal phase in evaluation workflow                      |

## Tips

* An opportunity is considered **open** when the current date is between `registrationFrom` and `registrationTo` and its status is `1`.
* Use `publishedRegistrations=EQ(true)` to find opportunities where results are public.
* Multi-phase opportunities have a parent opportunity. Filtra por `parent=EQ(@Opportunity:id)` to list phases.
* The `registrationCategories` and `registrationProponentTypes` arrays define what options applicants see on the registration form.
* Continuous-flow opportunities use a special date far in the future (`2111-01-01`) as `registrationTo`.
