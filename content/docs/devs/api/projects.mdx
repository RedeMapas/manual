> Endpoints da API para gerenciar projetos culturais no Mapas Culturais.

A **project** is a cultural initiative that groups related events together. Projects are owned by agents and can have a defined registration period. Events can be linked to projects, and projects can have child sub-projects.

Caminho base: `/api/project/`

<Info>
  Operações de leitura são públicas. Operações de escrita (`create`, `save`, `delete`) exigem autenticação com token JWT. Veja o [guia de autenticação](/docs/devs/auth/providers) para detalhes.
</Info>

## Endpoints

| Método   | Caminho                   | Descrição                |
| -------- | ---------------------- | -------------------------- |
| `GET`    | `/api/project/find`    | Listar ou buscar projects    |
| `GET`    | `/api/project/findOne` | Buscar um único project      |
| `POST`   | `/project/index`       | Criar um novo project       |
| `PATCH`  | `/project/single/{id}` | Atualizar um existente project |
| `DELETE` | `/project/single/{id}` | Delete a project           |

***

## GET /api/project/find

Retorna uma lista paginada de projects. Suporta filtros, ordenação, seleção de campos e busca textual.

### Parâmetros de consulta

**Parâmetros de controle** (prefixados com `@`)

<ParamField query="@select" type="string">
  Lista de campos separada por vírgula para incluir na resposta. Supports dot notation for related objects (e.g., `owner.name`).

  Exemplo: `@select=id,name,shortDescription,registrationFrom,registrationTo`
</ParamField>

<ParamField query="@page" type="integer" default="1">
  Número da página para resultados paginados. Começa em `1`.
</ParamField>

<ParamField query="@limit" type="integer" default="10">
  Número máximo de resultados por página. Use `-1` para desativar a paginação, com cautela em conjuntos grandes.
</ParamField>

<ParamField query="@order" type="string">
  Expressão de ordenação. Formato: `campo DIR`, em que `DIR` é `ASC` ou `DESC`.

  Exemplo: `@order=name ASC`
</ParamField>

<ParamField query="@keyword" type="string">
  Palavra-chave de busca textual. Pesquisa nos campos de nome e descrição.
</ParamField>

<ParamField query="@or" type="integer">
  Quando definido como `1`, combina todos os parâmetros de filtro com lógica `OR`, em vez do `AND` padrão.
</ParamField>

**Parâmetros de filtro**

<ParamField query="id" type="string">
  Filtra por project ID. Suporta operadores: `EQ(n)`, `IN(a,b,c)`, `BET(min,max)`, `!EQ(n)`.

  Exemplo: `id=EQ(4)` or `id=IN(1,2,3)`
</ParamField>

<ParamField query="name" type="string">
  Filtra por project name. Suporta operadores: `LIKE(pattern)`, `ILIKE(pattern)` (case-insensitive).

  Exemplo: `name=ILIKE(programa*)`
</ParamField>

<ParamField query="status" type="integer">
  Filtra por publication status.

  | Value | Meaning   |
  | ----- | --------- |
  | `1`   | Publicado |
  | `0`   | Rascunho     |
  | `-2`  | Arquivado  |
  | `-9`  | Desabilitado  |
  | `-10` | Lixeira     |

  Exemplo: `status=EQ(1)`
</ParamField>

<ParamField query="type" type="integer">
  Filtra por project type. Tipo IDs depend on instance configuration.

  Exemplo: `type=EQ(1)`
</ParamField>

<ParamField query="owner" type="string">
  Filtra por owner agent using entity reference syntax.

  Exemplo: `owner=EQ(@Agent:42)` returns projects owned by agent ID 42.
</ParamField>

<ParamField query="startsOn" type="string">
  Filtra por project start date. Supports date comparison operators.

  Exemplo: `startsOn=GTE(2024-01-01)`
</ParamField>

<ParamField query="endsOn" type="string">
  Filtra por project end date.

  Exemplo: `endsOn=LTE(2024-12-31)`
</ParamField>

### Resposta

Retorna um array JSON de project objects. The fields returned depend on `@select`.

<ResponseField name="id" type="integer" required>
  Identificador único de project identifier.
</ResponseField>

<ResponseField name="name" type="string" required>
  Project display name.
</ResponseField>

<ResponseField name="type" type="integer">
  Project type identifier. Defined by instance configuration.
</ResponseField>

<ResponseField name="shortDescription" type="string">
  Descrição curta (up to 400 characters).
</ResponseField>

<ResponseField name="longDescription" type="string">
  Descrição longa completa.
</ResponseField>

<ResponseField name="startsOn" type="string">
  Project start date (Data e hora em ISO 8601). When set, `endsOn` is also required.
</ResponseField>

<ResponseField name="endsOn" type="string">
  Project end date (Data e hora em ISO 8601). Must be greater than or equal to `startsOn`.
</ResponseField>

<ResponseField name="registrationFrom" type="string">
  Alias for `startsOn` in the context of event registration periods.
</ResponseField>

<ResponseField name="registrationTo" type="string">
  Alias for `endsOn` in the context of event registration periods.
</ResponseField>

<ResponseField name="status" type="integer">
  Status de publicação. See status values below.
</ResponseField>

<ResponseField name="createTimestamp" type="string">
  Data e hora em ISO 8601 when the project was created.
</ResponseField>

<ResponseField name="updateTimestamp" type="string">
  Data e hora em ISO 8601 of the last update.
</ResponseField>

<ResponseField name="owner" type="object">
  O agente proprietário deste project.

  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      ID do agente proprietário.
    </ResponseField>

    <ResponseField name="name" type="string">
      Nome do agente proprietário.
    </ResponseField>
  </Expandable>
</ResponseField>

### Exemplos

<CodeGroup>
  ```bash List published projects theme={null}
  curl "https://instance.example.com/api/project/find?@select=id,name,shortDescription&status=EQ(1)&@limit=20"
  ```

  ```bash Search projects by keyword theme={null}
  curl "https://instance.example.com/api/project/find?@select=id,name,registrationFrom,registrationTo&@keyword=cultura"
  ```

  ```bash Filter projects by owner agent theme={null}
  curl "https://instance.example.com/api/project/find?@select=id,name,owner.name&owner=EQ(@Agent:42)"
  ```

  ```bash Get projects ordered by creation date theme={null}
  curl "https://instance.example.com/api/project/find?@select=id,name,createTimestamp&@order=createTimestamp%20DESC&@limit=10"
  ```

  ```bash Get projects with type filter theme={null}
  curl "https://instance.example.com/api/project/find?@select=id,name,type&type=EQ(1)&status=EQ(1)"
  ```
</CodeGroup>

## GET /api/project/findOne

Retorna um único project. Aceita os mesmos parâmetros de filtro de `find`. Retorna o primeiro resultado encontrado.

```bash  theme={null}
curl "https://instance.example.com/api/project/findOne?@select=id,name,shortDescription,owner&id=EQ(4)"
```

## POST /project/index

Cria um novo project. Exige autenticação.

<Warning>
  Include a JWT token in the `Authorization` header. Add `MapasSDK-REQUEST: true` to receive JSON responses instead of HTTP redirects on write operations.
</Warning>

### Corpo da requisição

| Campo              | Tipo    | Obrigatório | Descrição                                                           |
| ------------------ | ------- | -------- | --------------------------------------------------------------------- |
| `name`             | string  | Sim      | Project name                                                          |
| `type`             | integer | Sim      | Project type ID                                                       |
| `shortDescription` | string  | Sim      | Descrição curta (máx. 400 caracteres)                               |
| `longDescription`  | string  | Não      | Descrição completa                                                   |
| `startsOn`         | string  | Não       | Start date in `YYYY-MM-DD` format. If set, `endsOn` is also required. |
| `endsOn`           | string  | Não       | End date in `YYYY-MM-DD` format. Must be >= `startsOn`.               |
| `terms`            | object  | Não       | Taxonomy terms, e.g. `{"area": ["Theater"]}`                          |

```bash  theme={null}
curl -X POST "https://instance.example.com/project/index" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -H "Content-Tipo: application/json" \
  -d '{
    "name": "Programa Cultural Verão",
    "type": 1,
    "shortDescription": "Programação cultural para o período de verão.",
    "startsOn": "2024-12-01",
    "endsOn": "2025-03-31"
  }'
```

## POST /project/single/\{id\}

Updates an existing project. Requires authentication and ownership (or admin role).

```bash  theme={null}
curl -X POST "https://instance.example.com/project/single/{id}" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -H "Content-Tipo: application/json" \
  -d '{
    "id": 4,
    "shortDescription": "Updated description."
  }'
```

***

## DELETE /project/single/\{id\}

Soft-deletes a project. Requires authentication and ownership (or admin role).

<Note>
  Deleting a project unlinks (but does not delete) events associated with that project.
</Note>

```bash  theme={null}
curl -X DELETE "https://instance.example.com/project/single/{id}" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -d 'id=4'
```

## Status values

| Value | Constant          | Meaning                         |
| ----- | ----------------- | ------------------------------- |
| `1`   | `STATUS_ENABLED`  | Publicado and publicly visible  |
| `0`   | `STATUS_DRAFT`    | Rascunho, visible only to owner    |
| `-2`  | `STATUS_ARCHIVED` | Arquivado                        |
| `-9`  | `STATUS_DISABLED` | Desabilitado by admin               |
| `-10` | `STATUS_TRASH`    | Deleted (soft delete, in trash) |

## Tips

* When `startsOn` is set, `endsOn` is mandatory and must be on or after `startsOn`.
* Projects can have child sub-projects. Filter with `parent=EQ(@Project:id)` to list sub-projects.
* Users with control over a project can create events within it. Other users can submit a workflow request to link their events.
* Use `@select` to limit response fields for better performance.
